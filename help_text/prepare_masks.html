This tool prepares and merges the masks that are used to compile a PaleoDEM and modify it. These masks can be in the form of polylines and polygons. Polylines are polygonized first, then all the invalid geometries in all the input layers are fixed. After this, the difference algorithm is run to remove the overlapping geometries. In this case, the order, in which the input layers are added in the table is important. The layers specified first (on top) are of higher order and the overlapping geometries in the underlying layers (lower order) will be removed by the difference algorithm.

The Input layer parameter (column):
Here the layers that need to be merged (prepared) should be specified. More rows for additional layers can be added, pressing the + button below the table. Inversely, the redundant rows can be deleted, pressing the x button.

The Category parameter:
Assign a category to the masks in the associated mask layer. The categories of each input layer is added to the category column of the resulting layer as attributes. The Category field is in edit mode and any arbitrary category can be added by typing the name of category in this field and pressing enter.

Specify the Output file path.
If there is no path specified here, the file will be created in the temporary folder. The full path will be shown in the Log tab and the result will be loaded to the map canvas.

Warnings:
I) Please avoid using these characters in the file name, as they might cause processing errors: ( ) / - % $ @ #
Note:If this tool is used repetitively with no path specified, previous results will be overwritten. To avoid this, specify a different path (or filename) each time
