How would you like to process the input layer:
Choose the processing tool you want to use.

Assign plate IDs partitions features into tectonic plates and assigns reconstruction plate IDs using pygplates.partition_into_plates. This is particularly useful for manually created vector layers, as it assigns the necessary plate IDs so they can later be used in the Reconstruct vector layers tool.

Input layer:
Choose a vector layer containing the features you want to assign plate IDs to.

Name of rotation model:
Select the plate reconstruction model to use. The rotation model and its associated static polygons will be downloaded automatically if not already cached locally.

Time of the input layer (in Ma):
The geological time (in millions of years ago) that the input features represent. The partitioning polygons will be reconstructed to this time before testing for overlap with the input features.

Partition method:
Split into plates — splits each feature into the partitioning plates it overlaps. If a feature overlaps two plates, it will be cloned twice, each clone receiving the geometry within the respective plate.
Most overlapping plate — assigns each feature to the single plate that most overlaps its geometry, without splitting. The feature geometry is left unmodified.

Output file path:
If there is no path specified here, the file will be created in the temporary folder. The full path will be shown in the Log tab and the result will be loaded to the map canvas.

Warning: Please avoid using these characters in the file name, as they might cause processing errors: ( ) / - % $ @ #
Note: If this tool is used repetitively with no path specified, previous results will be overwritten. To avoid this, specify a different path (or filename) each time
